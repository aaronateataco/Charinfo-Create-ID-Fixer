This tool resolves the **"Improper file format"** error ([Issue #16](https://github.com/Genwald/MiiPort/issues/16)) encountered when importing `.charinfo` files generated by third-party websites into MiiPort on the Nintendo Switch.

## The Problem
The Nintendo Switch's Mii validation system requires specific bits to be set in the **Create ID** (the 128-bit identifier for the Mii). Specifically, it expects the **UUIDv4 variant field** (bits 64 and 65 of the ID) to be set to `1` and `0` respectively. 

Many external Mii generators randomize these bits, resulting in a failure when attempting to import them to the console.

## The Solution
The provided script automatically performs an edit on the binary data of a `.charinfo` file:
1. It targets **offset `0x08`** (the 9th byte), which contains the variant bits.
2. It masks the existing value to ensure bit 7 is `1` and bit 6 is `0` (Variant 10).
3. It preserves all other Mii data (name, appearance, creator info) while making the file compliant with the Switch's requirements.

## How to Use

### Windows (Drag & Drop)
1. Ensure you have Python installed.
2. Drag and drop your `.charinfo` files directly onto `Fix-Miis.bat`.

### Command Line
Run the script manually by providing the file path(s) as arguments:

```bash
python mii_id_fixer.py <your_file>.charinfo
```

### Batch Processing
To fix every `.charinfo` file in a folder (PowerShell):
```powershell
python fix_charinfo.py (ls *.charinfo)
```
